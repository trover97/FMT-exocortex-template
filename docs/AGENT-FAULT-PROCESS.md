# Процесс учёта косяков агента

> **Почему:** Агент (Claude, Kimi, GPT) работает с ограниченным контекстом. После компактификации или между сессиями он «забывает» свои ошибки — и повторяет их. Пользователь тратит время на одни и те же поправки.
>
> **Решение:** Runtime memory о косяках агента. Агент читает свои ошибки перед сессией — как пилот чек-лист перед взлётом.

---

## Принцип

**Agent Fault Profile ≠ User Profile.**

| | User Profile | Agent Fault Profile |
|---|---|---|
| **Что** | Предпочтения пользователя | Ошибки агента |
| **Кто пишет** | Пользователь + агент (с согласия) | Пользователь или сам агент |
| **Кто читает** | Агент | Агент (себе!) |
| **Пример** | «Не люблю утренние звонки» | «Я часто заменяю 24-пунктовый чеклист на свой из памяти» |

---

## 5 шагов процесса

### Шаг 1. Обнаружение (в течение сессии)

**Кто:** Пользователь или агент (self-correction).

**Триггеры:**
- Пользователь говорит: «Ты пропустил X», «Это уже было», «Почему опять?»
- Агент сам замечает расхождение (stale data, пропущенный шаг протокола)

**Фиксация сразу:** короткая заметка в чате или mental note для Шага 2.

---

### Шаг 2. Фиксация (в конце сессии или в тот же день)

**Кто:** Пользователь (рекомендуется) или агент (если self-correction).

**Где:** `exocortex/feedback_YYYY-MM-DD-тема.md`

**Формат:**

```markdown
---
name: "Краткое название паттерна"
description: "В чём косяк, в каких ситуациях проявляется"
type: feedback
valid_from: YYYY-MM-DD
originSessionId: <id сессии>
---

## Правило N: Название

> Чёткая инструкция: что делать / что не делать.

**Журнал:**
- YYYY-MM-DD: контекст (WP-NNN, действие, последствия).
- YYYY-MM-DD: повторение, уточнение.
```

**Пример:**
```markdown
---
name: "Day Close — полный чеклист SKILL.md"
description: "Агент заменяет 24-пунктовый чеклист на упрощённый из памяти"
---

## Правило: Не выдумывать чеклист для R23

При верификации Day Close передавать sub-agent Haiku R23 полный текст чеклиста из SKILL.md строки 156-179, а не свою версию.

**Журнал:**
- 2026-04-24: Я составил 10-пунктовый чеклист вместо 24. Пропущены WakaTime, Memory Drift, apply-captures.
```

---

### Шаг 3. Синхронизация (раз в неделю или по триггеру)

**Кто:** Агент (автоматически или по команде).

**Команда:**
```bash
python3 scripts/sync_feedback_to_memory.py
```

**Что происходит:**
- Сканирует все `feedback_*.md` в `exocortex/` и `memory/`
- Извлекает правила и журналы
- Считает частоту повторений (trust_score)
- Пишет в SQLite: `exocortex/agent-fault-profile/iwe_memory.db`

**Идемпотентность:** повторный запуск не дублирует записи.

---

### Шаг 4. Напоминание (перед каждой сессией)

**Кто:** Агент (автоматически по protocol-open).

**Команда:**
```bash
bash scripts/agent_fault_remind.sh close   # или open / work
```

**Что происходит:**
- SQLite выдаёт топ-3 косяка по trust_score
- 🔴 (≥0.8) = критический, часто повторяется
- 🟡 (0.65–0.79) = внимание
- 🟢 (<0.65) = на заметку

**Агент встраивает напоминания в system prompt** — как липкую записку.

---

### Шаг 5. Ретро (раз в неделю, в Week Close)

**Кто:** Пользователь + агент.

**Вопросы:**
1. Какие косяки повторились на этой неделе?
2. Есть ли новые паттерны, которых ещё нет в feedback?
3. Какие 🔴 косяки опустились до 🟡 (агент научился)?

**Действие:**
- Новые косяки → Шаг 2 (фиксация)
- Исправленные косяки → добавить `superseded_by` в frontmatter feedback-файла

---

## Скрипты

| Скрипт | Назначение | Зависимости |
|--------|-----------|-------------|
| `sync_feedback_to_memory.py` | Парсит feedback → SQLite | Python 3, sqlite3 (stdlib) |
| `agent_fault_remind.py` | Выдаёт топ-N напоминаний | Python 3, sqlite3 (stdlib) |
| `agent_fault_remind.sh` | Wrapper для удобства | bash |

**Zero external dependencies.** Работает на любом Mac/Linux с Python 3.

---

## Метрики эффективности

Как понять, что процесс работает:

| Метрика | Как измерить | Цель |
|---------|-------------|------|
| **Полнота чеклиста** | Сравнить 3 сессии с remind vs 3 без | С remind ≥ на 30% меньше пропусков |
| **Время на поправки** | Засечь, сколько минут пользователь тратит на «ты опять...» | <5 мин/сессия |
| **Количество 🔴 косяков** | `agent_fault_remind.py --stats` | Не растёт 2 недели подряд |
| **Trust score косяка** | Динамика по конкретному правилу | Падает после фиксации |

## Интеграция с ОРЗ

**Перед protocol-open:**
```bash
bash scripts/agent_fault_remind.sh open
```
→ Вставить вывод в system prompt.

**После protocol-close:**
```bash
# Если были косяки — записать их через checklist_memory.py
# или вручную создать feedback-файл
```

**В Week Close:**
```bash
python3 scripts/sync_feedback_to_memory.py
bash scripts/agent_fault_remind.sh --stats
```
→ Обсудить динамику на стратегической сессии.

## Чеклист внедрения

- [ ] Создать первый feedback-файл (фиксация косяка из последней сессии)
- [ ] Запустить `sync_feedback_to_memory.py` (проверить, что SQLite создалась)
- [ ] Запустить `agent_fault_remind.sh work` (увидеть топ-3)
- [ ] Провести 1 сессию с напоминаниями (проверить, что агент их видит)
- [ ] Через 5 дней: smoke-test (3 с remind vs 3 без)
- [ ] Через 2 недели: ретро + решение о доставке в шаблон
