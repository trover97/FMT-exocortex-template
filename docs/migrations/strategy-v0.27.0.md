# Миграция: Strategy.md / WeekPlan под v0.27.0

> Применимо к пользователям, которые установили шаблон **до v0.27.0** и имеют свой `DS-strategy/docs/Strategy.md` от прежней версии.

## Что изменилось в v0.27.0

В `seed/strategy/docs/Strategy.md` добавлены три блока (см. CHANGELOG `[0.27.0]`):

1. **«Состояние месяца — фаза стратегической позиции»** — 4 фазы (Развитие / Хаос / Потолок / Пивот), диагностика по 5 сигналам, playbook на каждую, сигналы перехода. Источник: PD.FORM.078, WP-196 Ф12.1.
2. **«Калибр личности»** — gap-analysis по 3 направлениям (горизонт / bus factor / публичность). Источник: PD.CHR.007.
3. **Строка-источник «НЭП-триады»** перед таблицей R1-R{N}.

Strategy Session теперь начинается с явной декларации фазы и playbook'а под неё, а не с произвольного выбора РП.

В `memory/templates-dayplan.md` добавлены: критерии отбора РП, ТОС недели, time-boxing.

## Что `update.sh` делает

`update.sh` **не трогает** ваш `DS-strategy/` (это личное планирование, защищено от перезаписи). Поэтому новые блоки v0.27.0 в ваш `Strategy.md` **не добавятся автоматически**.

## Как мигрировать (≤15 мин)

1. Откройте `seed/strategy/docs/Strategy.md` в FMT-exocortex-template.
2. Откройте свой `DS-strategy/docs/Strategy.md`.
3. Скопируйте недостающие блоки в правильное место:
   - Секция «Состояние месяца» — между «Видение» и «Цели на горизонт».
   - Секция «Калибр личности» — после «Целей на горизонт», перед НЭП-триадой.
   - Строка-источник «НЭП-триады» — над таблицей R1-R{N}.
4. На ближайшей Strategy Session явно продекларируйте текущую фазу (Развитие / Хаос / Потолок / Пивот) и выберите playbook.

## Что не нужно мигрировать

- `Dissatisfactions.md`, `Session Agenda.md`, `WP-REGISTRY.md` в `seed/` — это пустые шаблоны, не обновлялись содержательно в v0.27.0.
- WeekPlan — динамический файл, новые шаблонные блоки попадут на ближайшей session-prep.
