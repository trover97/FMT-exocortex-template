# Перенос знаний в эту установку (offline / Windows)

> Главное различение IWE: **форма vs знания**.
> Этот репозиторий (FMT-шаблон) — *форма*: структура, скрипты, скиллы, протоколы.
> Твоих личных знаний внутри него нет — они живут отдельно. Поэтому «перенести
> знания» = положить рядом свои данные, а не переписать шаблон.

## Где какие знания лежат

| Знание | Где физически | Часть шаблона? |
|--------|---------------|----------------|
| Архивные РП, WP-REGISTRY, планы недель, inbox | репо `DS-strategy` (governance-хаб) | ❌ отдельно |
| Паки (предметные знания) | репо `PACK-*` | ❌ отдельно |
| Командные планы | `DS-ecosystem-development` и др. `DS-*` | ❌ отдельно |
| Авто-память агента | `~/.qwen/projects/<проект>/memory/` | ❌ вне репо |
| Накопленные факты IWE (MEMORY.md, протоколы) | `memory/*.md` в каталоге установки | ✅ в шаблоне |
| Твои настройки | `params.yaml`, `extensions/` | ✅ (обновление не трогает) |
| Твои правила | секция §9 «Авторское» в `QWEN.md` | ✅ |

Раскладка (как в оригинальном IWE — workspace = внешняя папка):
```
~/IWE/                          ← workspace = IWE_ROOT
├── FMT-exocortex-template/     ← шаблон (этот репо)
├── QWEN.md  .qwen/  memory/    ← развёрнуто из FMT через setup-offline.sh
├── DS-strategy/                ← твои РП, планы, архив (положи сюда)
├── PACK-self-development/       ← твои паки (положи сюда)
└── PACK-.../
```
Хуки резолвят корень через `IWE_ROOT` (выставляется в `~/.bashrc`); соседние
репо они находят относительно него.

## Сценарий А — у тебя уже есть DS-strategy и PACK-* отдельными папками

Переносить почти нечего:
1. Скопируй каталоги `DS-strategy/` и `PACK-*/` внутрь каталога установки (рядом с `QWEN.md`).
2. Они работают и с Qwen — формат знаний от агента не зависит.
3. Перенеси авто-память (см. ниже).

## Сценарий Б — всё свалено внутри одного старого FMT-репо

Распакуй новую ветку в новый каталог, затем перенеси ТОЛЬКО свои данные
(не форму) из старого в новый:

```bash
# свои данные:
cp -r СТАРЫЙ/DS-strategy        НОВЫЙ/    # если был внутри
cp -r СТАРЫЙ/PACK-*             НОВЫЙ/
cp    СТАРЫЙ/memory/MEMORY.md   НОВЫЙ/memory/     # свои факты (не перезатирай протоколы вслепую)
cp    СТАРЫЙ/params.yaml        НОВЫЙ/
cp -r СТАРЫЙ/extensions/*       НОВЫЙ/extensions/
# свою секцию §9 в QWEN.md перенеси вручную (diff)
```

## Как Qwen определяет каталог памяти

`<база>/projects/<id>/memory/`, где:
- **база** = `~/.qwen` (или `QWEN_HOME` / `QWEN_RUNTIME_DIR`, если заданы);
- **id** = `sanitizeCwd(каталог-запуска-qwen)`: на Windows весь путь в нижний
  регистр, затем любой символ кроме `[A-Za-z0-9]` → `-`. Путь берётся в
  Windows-форме (`C:\Work\IWE`), которую видит Node.

Примеры (запуск qwen из workspace):
- `C:\Work\IWE` → `~/.qwen/projects/c--work-iwe/memory/`
- `C:\Users\John.Doe\IWE` → `~/.qwen/projects/c--users-john-doe-iwe/memory/`

`setup-offline.sh` вычисляет этот id автоматически (через `cygpath -w` + lower +
замена), кладёт туда память IWE и симлинкует `workspace/memory` на неё.
**Проверь после первого запуска** `qwen`: `ls ~/.qwen/projects/` — если фактическое
имя отличается, скопируй: `cp <вычисленный>/*.md ~/.qwen/projects/<факт>/memory/`.

## Авто-память (claude → qwen)

Если раньше был Claude Code, накопленная память лежит в
`~/.claude/projects/<slug>/memory/`. Перенеси в qwen-каталог (id см. выше):

```bash
# 1) запусти qwen один раз — он создаст ~/.qwen/projects/<id>/memory/
# 2) затем (подставь фактический id из ls ~/.qwen/projects/):
cp ~/.claude/projects/*/memory/*.md ~/.qwen/projects/<id>/memory/
```

## Дальнейшие обновления формы (git локальный)

```bash
cd ~/IWE
git add -A && git commit -m "снимок перед обновлением"
# распакуй новый ZIP формы поверх → git status покажет дельту
git add -A && git commit -m "обновление формы"
git log --oneline    # история = страховка для отката
```
Свои данные (`DS-strategy`, `PACK-*`, `memory/MEMORY.md`) при этом не трогаются.
