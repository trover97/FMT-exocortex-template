# Различения (ключевые термины)

> Загружается всегда. Источник: CLAUDE.md § Различения (вынесено для сжатия контекста).

- **IWE = Intellectual Work Environment** (НЕ «Intelligent», НЕ «Working»). Аналогия: IDE.
- **AUTHOR-ONLY зоны** в `memory/protocol-*.md`: `<!-- AUTHOR-ONLY -->...<!-- /AUTHOR-ONLY -->` — авторский контент, не затирать при sync.
- **strategy_day:** из `day-rhythm-config.yaml`. В этот день: session-prep вместо day-plan, DayPlan не создаётся.
- **Ru-first (SPF §5 п.13):** Русский — основной язык. EN только для: YAML-ключей, аббревиатур, имён собственных.
- **FAQ:** Доменный → `DP.IWE.002 §11`. Практический → `LEARNING-PATH §11`.
- **Скилл ≠ Роль ≠ Протокол.** Скилл = точечное действие. Роль = контекст на сессию. Протокол = ритуал ОРЗ.
- **Развитие ≠ Образование.** IWE про развитие. Не использовать: «школа», «вуз», «студент», «преподаватель».
- **settings.json ≠ settings.local.json.** Проектный (hooks, git) vs персональный (permissions, .gitignore).
- **Day Open (VS Code) ≠ DayPlan (git).** Compact dashboard vs полный отчёт.
- **Проектировать роль агента в контексте**, не «проектировать агента» (DP.D.025).
- **OAuth через бота**, не ручные ключи. Все интеграции — через OAuth.
- **«Настрой календарь»** → `bash setup/optional/setup-calendar.sh`.
- **Pack-знание ≠ Реализационное решение** (HD #29): доменная истина → Pack. Технический выбор → DS.
- **Персонализация ≠ Индивидуализация ≠ Адаптивность (DP.CONCEPT.003, HD #49).** Три механизма адаптивной персонализации. Персонализация = платформа подстраивает контент под ЦД (runtime, Портной). Индивидуализация = пользователь настраивает среду сам (Personal IWE, Pack). Адаптивность = система меняет поведение по обратной связи во времени (BKT, HLR). Путать = чинить не тот слой при сбое.
- **OwnerIntegrity:** Один факт — одно место. Если факт в Pack, не дублировать в DS (и наоборот). Дублирование = ошибка синхронизации.
- **Рабочий документ ≠ Публикация.** Рабочий документ (протокол, CLAUDE.md, Pack): точность > стиль. Публикация (пост, руководство): стиль = доверие. ИИ пишет оба, но публикация требует редактуры «голосом автора». Антипаттерны → DP.SC.005.
- **Экзоскелет ≠ Автопилот (DP.D.046).** Экзоскелет = усиление мышления, human gate обязателен (Pack, стратегия). Автопилот = LLM автономно (разведка, черновой слой, linting). Смешение: автопилот на Pack → некритические знания; экзоскелет на черновиках → бутылочное горлышко.
- **Первое решение ≠ лучшее решение.** Для любой технической проблемы — сначала диагностика первопричины, потом решение. Костыль (обход симптома) предлагать ТОЛЬКО если: (1) системное решение явно невозможно, или (2) пользователь прямо просит быстрый фикс. Иначе — системное решение первым, костыль — как запасной вариант с пометкой «временно».
