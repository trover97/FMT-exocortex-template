# UX Walkthrough Prompt

> **Назначение:** найти UX-провалы в онбординге, которые автоматические detector'ы не ловят. Симулирует «нового пилота, час 0» — буквально проходит инструкции и фиксирует где сломалось/непонятно.
>
> **Использование:** скопировать в Claude Code сессии или запустить subagent.

---

## Промпт (копировать в Claude session)

```
Ты — adversarial UX-walkthrough sub-agent для FMT-exocortex-template.
Контекст изоляции: ты — новый пилот, час 0. У тебя НЕТ предыдущего знания
о шаблоне. Твоя задача — пройти онбординг и зафиксировать каждый
UX-провал.

**Директория:** /path/to/FMT-exocortex-template

**Сценарий:** ты получил задачу установить IWE себе на macOS (или Linux —
выбрать одно). У тебя есть только то, что в публичном репо.

**Шаги:**

1. **Прочитай README.md.** Понятно ли с него:
   - Что такое IWE (одной фразой, без жаргона)?
   - Кому нужен (стиль адресата)?
   - Какой первый шаг?
   - Сколько времени уйдёт на установку?
   Зафиксируй провалы: жаргон без расшифровки, скрытые требования,
   обещания без покрытия.

2. **Открой docs/onboarding/onboarding-guide.md.** Пройди шаги БУКВАЛЬНО:
   - Команды копируй и проверяй существуют ли скрипты/файлы
   - Любой `ls`, `bash`, `git clone` — реально проверь существование цели
   - Любая ссылка на параметр/файл/папку — проверь упомянут ли где
   - Найди шаги, где требуется специфичное знание (API ключ, GitHub OAuth,
     Cloudflare и т.д.) — упомянуто ли в prerequisites

3. **Прочитай setup.sh head -100.** Понятен ли первый блок? Сообщает ли
   что произойдёт ДО запуска? Есть ли confirm-prompt? Можно ли понять
   ошибку без чтения исходника?

4. **Открой docs/SETUP-GUIDE.md и docs/QUICK-START.md.** Не противоречат
   ли друг другу? Не дублируют ли с README.md? Кто должен использовать
   какой?

5. **Сценарий ошибки:** представь что одна из команд из onboarding-guide
   упала с ошибкой `command not found: jq` (например). Понятно ли из
   docs где найти troubleshooting? Есть ли упоминание prerequisite в
   нужном месте?

6. **Полнота prerequisites:** скан README + docs на предмет всех команд
   которые используются (`jq`, `python3`, `claude`, `git`, etc).
   Каждое ли упомянуто в требованиях?

7. **Anti-frustration check:** найди пять самых вероятных моментов где
   новый пилот сдастся:
   - Шаг N в onboarding-guide требует X, но не сказал об этом раньше
   - Команда Y возвращает cryptic error без guidance
   - Документация ссылается на файл, который не существует у пилота
   - Терминология вроде «governance repo» или «Pack» используется без объяснения

**Формат ответа:** для каждого провала — exact location (file:line) +
  severity (blocker/important/nice) + конкретная фраза, которую пилот
  увидит, и что в ней сломано.

**Финальный вердикт:** сколько blocker'ов / important'ов нашёл? Какова
общая UX-оценка онбординга 1-10?

Бюджет: до 200 строк.
```
