# WP-REGISTRY — реестр рабочих продуктов

> **Назначение:** один источник правды по всем твоим РП (рабочим продуктам). Открыт всегда. Update при создании, продвижении, закрытии РП.

## Легенда статусов

| Эмодзи | Статус | Что значит |
|---|---|---|
| 🔄 | in_progress | Активный РП, есть прогресс на этой неделе |
| ⏳ | pending / backlog | Запланирован, ещё не начат |
| ✅ | done | Завершён, артефакт принят |
| ↗️ | merged-into | Поглощён другим РП (с указанием куда) |
| 📦 | archived | Снят с приоритета, не закрыт формально |

## Правила работы с реестром

1. **Нумерация — последовательные целые** (1, 2, 3 …). Без буквенных суффиксов.
2. **Название РП — существительное-артефакт**, не глагол. ✅ «Дизайн системы X». ❌ «Сделать систему X».
3. **Закрытый РП:** строка зачёркивается `~~...~~` целиком (см. пример). Эмодзи статуса ✅ — БЕЗ тильд.
4. **Синхронизация при переименовании:** обновить одновременно — REGISTRY + WeekPlan + DayPlan (если активен) + WP-context file в inbox/.
