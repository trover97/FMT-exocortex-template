# FMT-strategy-template — шаблон личного стратегического хаба

> **Назначение.** Минимальный scaffold персонального DS-strategy репо для пилота IWE. Из него начинается твоя личная работа со стратегией: миссия, неудовлетворённости, рабочие продукты (РП), недельные планы.

## Что внутри

| Папка / файл | Назначение |
|---|---|
| `CLAUDE.md` | Инструкции для Claude Code в этом репо. Роль — Стратег. |
| `docs/` | Долгоживущие документы: Strategy, Dissatisfactions, WP-REGISTRY |
| `current/` | Регулярно обновляемые: WeekPlan W{N}, DayPlan YYYY-MM-DD |
| `archive/` | Старые планы и отчёты после Week Close / Month Close |
| `inbox/` | Входящие заметки, черновики, fleeting-notes, WP-context файлы |

## Как использовать

1. **Склонируй или скопируй этот шаблон** в `~/IWE/DS-strategy/` (или другое имя на твой вкус — важно, чтобы префикс был `DS-`).
2. **Проинициализируй git:**
   ```bash
   cd ~/IWE/DS-strategy
   git init && git add . && git commit -m "init from FMT-strategy-template"
   ```
3. **Заполни заготовки в `docs/`:**
   - `Strategy.md` — твои направления развития (R1, R2, …) с бюджетами времени
   - `Dissatisfactions.md` — список «что меня не устраивает» (источник для будущих РП)
   - `WP-REGISTRY.md` — реестр рабочих продуктов (вначале пустой)
4. **Начни первый стратегический цикл** — попроси Claude Code: «проведём первую стратегическую сессию» (скилл `/strategy-session`).

## Связь с остальной IWE

- **Личное знание:** `~/IWE/PACK-personal/` (Lifework, ценности, миссия)
- **Платформенное знание:** `~/IWE/PACK-digital-platform/` (общая онтология)
- **Командные планы:** `~/IWE/DS-ecosystem-development/WORKPLAN.md`

## Принципы (из CLAUDE.md)

1. **РП = артефакт с критериями** — не описание, не намерение
2. **Учёт ≠ планирование** — факты (коммиты) отдельно от намерений (РП)
3. **Work-Product Gate** — нетривиальная работа (>15 мин) только при наличии РП в плане недели
4. **Не хранить здесь предметные знания** — они в Pack
