# Appendix: Срез ЦД + Guide для сессии (session-context)

> Артефакт `context/session-context-YYYY-MM-DD.md` готовится автоматизированным шагом session-prep **до** интерактивной сессии и потребляется шагами 04 (стратегическая сверка) и 05a (калибр-замер). Назначение — дать Стратегу актуальный срез цифрового двойника (ЦД) и персонального руководства одним файлом, без обращения к БД в ходе диалога.

## Предусловие

Готовится **только** при согласии на трекинг (`learning.tracking_consent = granted`, см. [consent-check.md](consent-check.md)). При `denied` / `null` файл **не создаётся**; шаги 04/05a идут в degraded-режиме.

## Формат артефакта

```markdown
---
generated_at: <ISO-8601 UTC>
generated_by: session-prep
pilot: <идентификатор пилота>
consent: granted
dt_snapshot_date: <YYYY-MM-DD — дата среза ЦД>
schema_version: 1
---

# Срез сессии <YYYY-MM-DD>

## 1. Цифровой двойник (производные агрегаты)
- **Ступень (cp-профиль):** cp.rhy / cp.wld / cp.skl / cp.iwe / cp.int / cp.agt — по FORM.089 §6.1
- **Bottleneck:** <слот с минимальным индексом>
- **Мультипликатор недели:** <значение + источник: WakaTime/slot_logged>
- **Инвестированное время за период:** <часы> (только саморазвитие, не учтённое)
- **Фаза:** Хаос / Потолок / Поворот / Развитие

## 2. Срез персонального руководства (Guide)
- **Программа:** ЛР / РР / ИР
- **Текущий фокус-домен:** <из personal-guide>
- **Открытые рекомендации:** <до 3 пунктов>

## 3. Сигналы для стратегирования
- Расхождения «план vs факт» за период
- Триггеры стоп-листа РП (если сработали)
- Незакрытые carry-over прошлой недели
```

## Правила

1. **Derived, не источник.** Файл — снимок; источник истины ЦД — Neon (Память.Derived). Не редактировать вручную в ходе сессии.
2. **Один файл на дату.** `session-context-YYYY-MM-DD.md`. Повторный запуск session-prep в тот же день — перезапись.
3. **Срок годности.** Старше 1 дня → шаг 04 помечает «срез устарел», предлагает пере-prep или degraded.
4. **Отсутствие файла ≠ ошибка.** Нет файла → шаги 04/05a идут на самодекларации (как при degraded). Не блокировать сессию.
5. **PII-граница.** В срез попадают только агрегаты (числа, ступень, фаза), не сырьё переписки/рефлексий (B7.3).
