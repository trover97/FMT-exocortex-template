# Appendix: Проверка согласия на трекинг

> Единый источник истины согласия — таблица `learning.tracking_consent` (Neon). Frontmatter WeekPlan/DayPlan **не** является источником согласия (drift). Перед любой записью трекинга или чтением производных цифрового двойника (ЦД) шаг сверяется здесь.

## Когда вызывать

| Шаг | Зачем |
|-----|-------|
| 04 (стратегическая сверка) | Перед чтением `context/session-context-*.md` (срез ЦД/Guide) — данные производны от трекинга |
| 05a (калибр: замер) | Перед использованием метрик мастерства из ЦД |
| 05c (калибр: трекинг) | Перед **записью** в трекеры / domain_event |

## Состояния согласия

`learning.tracking_consent` для пилота возвращает одно из:

| Состояние | Значение | Поведение шага |
|-----------|----------|----------------|
| `granted` | Согласие дано | Полный режим: читать ЦД-срез, писать трекинг, считать метрики |
| `denied` | Явный отказ | **Degraded-режим:** не читать ЦД, не писать трекинг/domain_event. Сессия идёт на самодекларации пилота |
| `null` / отсутствует | Согласие не запрашивалось | Трактовать как `denied` (fail-safe). Однократно предложить пройти `/consent` — без давления |

## Degraded-режим (denied / null)

1. **Не читать** `context/session-context-YYYY-MM-DD.md` (он производен от трекинга). Если файл существует — пропустить с пометкой «ЦД-срез недоступен: нет согласия».
2. **Не писать** в `Caliber-artifacts-tracker.md` поля, опирающиеся на трекинг; калибр-линза идёт на самооценке пилота, без записи domain_event.
3. **Не вычислять** мультипликатор/ступень из ЦД — спрашивать пилота напрямую.
4. **Сообщить пилоту один раз** (без повтора в той же сессии):
   > «Трекинг выключен (нет согласия). Веду сессию по твоим словам, без чтения цифрового двойника. Включить трекинг — команда `/consent`.»
5. Сессия **не блокируется** — стратегирование и планирование возможны полностью на самодекларации.

## Проверка (как именно)

- В Claude Code / VS Code: чтение `learning.tracking_consent` через MCP/скрипт. Нет доступа к Neon → трактовать как `null` (degraded).
- В браузерном канале: согласие приходит в контексте онбординга; отсутствие → degraded.

**Инвариант:** одно состояние согласия на сессию. Не перепроверять между шагами — зафиксировать в начале (шаг 00/04) и переиспользовать.
