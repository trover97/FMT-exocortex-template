# Step 02: Inbox Triage

## ВДВ

| | |
|---|---|
| **Вход** | `inbox/fleeting-notes.md` + свежие файлы за неделю + QA-отчёт бота |
| **Действие** | Классифицировать каждый элемент: в план / capture / удалить / отложить |
| **Выход** | Решения по inbox, обновлённый fleeting-notes.md |

## Инструкция для Claude

1. **Прочитать источники:**
   - `inbox/fleeting-notes.md`
   - Все файлы из `inbox/` (кроме .DS_Store и .docx)
   - QA-отчёт бота: `DS-agent-workspace/scheduler/feedback-triage/` (последний по дате)

2. **По каждому элементу** пилот решает:
   - **В план недели** → добавить в candidate pool (шаг 06a)
   - **Capture в Pack** → зафиксировать через `/ke` или записать в Pack
   - **Удалить** → применить сразу
   - **Отложить** → пометить `🔄` с датой пересмотра

3. **Недельная агрегация Inbox Triage:**
   - Собери все `🔄` заметки (идеи, висящие >7 дней)
   - Собери непринятые предложения из прошлых Note-Review
   - Сформируй 3 корзины: ✅ Архив / 📌 Предложения в РП / ❓ На решение

4. **Вопрос пилоту:**
   > «Продолжить к НЭП / скорректировать / пропустить?»
