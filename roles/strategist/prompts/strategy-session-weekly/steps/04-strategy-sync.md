# Step 04: Стратегическая сверка

## ВДВ

| | |
|---|---|
| **Вход** | `docs/Strategy.md` + данные MonthClose (если gate сработал в шаге 00) + `context/session-context-YYYY-MM-DD.md` из session-prep |
| **Действие** | Сверить стратегию с текущими РП, обновить Strategy.md по решениям пилота |
| **Выход** | Корректировки Strategy.md, расхождения зафиксированы |

## Инструкция для Claude

1. **Прочитать:**
   - `docs/Strategy.md` — фокусы года, Q1 цели, приоритеты месяца
   - `context/session-context-YYYY-MM-DD.md` (если есть — DT + Guide срез из session-prep; формат и предусловие согласия — [../appendix/dt-guide-prep.md](../appendix/dt-guide-prep.md)). При denied/null согласии файл отсутствует → degraded ([../appendix/consent-check.md](../appendix/consent-check.md)).
   - MonthClose-сводку (если шаг 00 сработал)

2. **Если первая сессия месяца:**
   - Сверь фокус стратегии с данными MonthClose (мультипликатор, фаза, итоги R, инсайты)
   - Что подтвердилось, что отвергнуто, что перенеслось

3. **Обсудить с пилотом:**
   - Расхождения между стратегией и текущими РП
   - Корректировки Strategy.md

4. **Связь R→P (если есть Projects.md):**
   - Каждый R-результат месяца должен ссылаться на проект P
   - Проверь колонку `P` (P1-P6)
   - Расхождения → пересмотр привязки

5. **Вопрос пилоту:**
   > «Продолжить к линзе калибра / скорректировать / пропустить?»
