# Step 05b: Калибр — рекомендация

## ВДВ

| | |
|---|---|
| **Вход** | Измерения из шага 05a (C_vis, C_act, разрыв, пакет C_act) |
| **Действие** | Объявить рекомендацию роста, предложить РП |
| **Выход** | Решение пилота: создать РП / remind_monthly / dismiss |

## Инструкция для Claude

1. **Объяви рекомендацию уровня C_act+1** (если двойной gate из 05a пройден)

2. **Логика напоминания** (читая трекер):
   - `proposed` 1-я сессия → объяви рекомендацию как РП
   - `proposed` 2-я сессия → напомни: «всё ещё в зоне роста, поставим РП?»
   - `proposed` 3-я сессия → спроси частоту: создать / `remind_monthly` / `dismissed`
   - `remind_monthly` >4 недель → напомни «месяц назад: [X], изменилось ли?»
   - `dismissed` → не напоминать (возврат только при изменении C_vis/C_act)
   - `remind_monthly` >12 мес → авто-`dismissed`

3. **Не более одного up-level артефакта в фокусе**
   - Если в трекере уже есть `proposed`/`in-progress` уровня выше C_act — НЕ добавляй второй
   - Анти-имитация (FORM.091 §8 «Подмена видения созданием»)

4. **Антипаттерн:** «У тебя S0 в НЭП → давай напишем Конституцию Дела». Корректно: рекомендации идут от C_act+1, не от C_vis.

5. **Вопрос пилоту:**
   > «Продолжить к трекингу калибра / скорректировать / пропустить?»
