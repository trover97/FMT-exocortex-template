# R-вопросник

> **Класс R (Review/judgment):** вопросы для переосмысления. **Не yes/no.** Цель — поймать сигналы, которые не ловит чеклист и drift-скрипт.
> Различение T ≠ S ≠ R: см. `.claude/rules/distinctions.md`.
> Ответы фиксируются в Week Report / Month Report как отдельный раздел «R-ответы».

## Week Close — 3 вопроса

> Проводить в конце недели. Время: 3-5 мин.
> **Принцип:** Конец недели = усталость. 7 вопросов → формализм. 3 вопроса + данные от агента → честный сигнал.
> **Алгоритм агента перед вопросами:** (1) запустить `behaviour-report.sh`, (2) прочитать incident-log текущего месяца, (3) предъявить паттерны пользователю — он подтверждает, не вспоминает сам.
> **Изменено:** 13 апр 2026 (сокращено с 7 до 3 по обратной связи).

1. **Вот что система пропускала регулярно — [агент вставляет факты из Behaviour Report + incidents]. Это системный паттерн или разовое?**
   _Цель: выявить мёртвые правила или точки, где нужен детектор. Если паттерн 3 недели подряд → правило удалить или детектор создать._

2. **Какой WP взял больше времени, чем планировалось — и почему одним словом?**
   _Цель: калибровать оценки. Один WP, одно слово: «недооценил» / «блокер» / «расширился»._

3. **Если бы мог отменить одно действие этой недели — что?**
   _Цель: честный стоп-сигнал. «Нечего» 3 недели подряд → вопрос снять._

## Month Close — 6 доп. вопросов

> Проводить в первый Пн месяца через скилл `/month-close` (шаг 6 алгоритма). Время: ~10 мин.
>
> **Правило заполнения:** агент заполняет M1, M2, M3, M6 АВТОМАТИЧЕСКИ из данных ДО показа пользователю. Пользователь только подтверждает или корректирует. Только M4 и M5 — субъективные вопросы пользователю. Без английских терминов («dormant» → «спящий» / «на паузе»).

### Заполняются агентом из данных (без вопроса пользователю)

M1. **Какое различение я добавил в этом месяце — и использовал ли его хоть раз?**
    _Источник: `git log --since="YYYY-MM-01" .claude/rules/distinctions.md` + `memory/hard-distinctions.md`. Агент находит новые HD, проверяет упоминания в commit messages / РП-контекстах того же месяца. Цель: decommission для неиспользованных distinctions (3 месяца без использования → архив)._

M2. **Какой файл memory стал длиннее всех за месяц? Сигнал роста или гниения?**
    _Источник: `wc -l memory/*.md` + git diff за месяц. Агент сам различает рост (новые HD по горячим следам) и гниение (дубликаты, уточнения уточнений)._

M3. **Какая метрика здоровья (мультипликатор, drift, memory-bleed) ухудшилась? Что её толкает?**
    _Источник: `multiplier-trend.md` (динамика), `iwe-drift.sh` (drift), `memory-bleed.sh` (memory). Агент сравнивает с прошлым месяцем и называет фактор._

M6. **Перевод в «спящий» режим: какие T-действия — кандидаты?**
    _Источник: `iwe-drift.sh --activity` читает `sync-manifest.yaml § activity_checks:` и выдаёт список действий, не сработавших 3 окна подряд. Плюс ручные кандидаты для действий без измеримого следа. Если кандидатов нет — так и пишем «нет», пользователю не задаём вопрос. Если есть — показываем список и пользователь решает (active / спящий / архив)._

### Спрашиваются у пользователя (субъективные)

M4. **Какое правило/скилл я НЕ добавил, хотя думал об этом? Почему?**
    _Цель: отделить «забыл» (проблема capture) от «не нужно» (сигнал «система способна остановить рост»)._

M5. **Что в IWE сработало неожиданно хорошо? Что закрепить как правило?**
    _Цель: позитивный кейс. Промоция случайного успеха в регламент (staging → FMT). Без этого вопроса позитивные паттерны теряются._

## Формат ответов

- Один абзац на вопрос (2-5 предложений).
- «Нечего ответить» — валидный ответ (3 раза подряд → вопрос кандидат на удаление из регламента).
- Агент может предлагать факты (из drift-отчёта, git log, WakaTime), но не формулировать ответ за человека.

## Антипаттерны

- **Формальное закрытие:** «всё ок, нечего отметить». Если каждая неделя так — R-вопросник мёртв, надо менять вопросы.
- **Декларация планов:** «в следующую неделю исправлю». R-вопросник про прошлое, не про будущее. Выводы — отдельная строка в отчёте.
- **Самобичевание:** «я плохо работал». Цель — найти сигнал в системе, не оценку себя.
